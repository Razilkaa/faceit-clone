# Pet project: свой FACEIT (с нуля, своими руками)

Полноценный клон ключевых фич FACEIT. Backend на Python (FastAPI), frontend на vanilla JS + Node (тонкий Express-слой для статики и dev-прокси), реальные данные из FACEIT Data API и Twitch Helix, Docker для запуска.

**Главное правило обучения:** код пишешь ты. Я ревью, намёки, объяснение концептов. Без `solution` я готовый код не присылаю.

**Главное правило мотивации:** каждая версия закрывается **видимым результатом на странице**. Бэк без фронта не закрываем — иначе скучно и забросишь.

---

## Правила работы

1. Код пишешь ты, я не пишу за тебя.
2. Каждую строку, которую написал, ты должен уметь объяснить вслух.
3. Когда застрял — пиши режим:
   - `review` — проверь, найди слабые места.
   - `hint-1` — крошечный намёк (одно слово, одна функция).
   - `hint-2` — объяснение концепта на пальцах.
   - `hint-3` — псевдокод, без готового решения.
   - `solution` — полный код + разбор. Только если попросил явно.
4. Ошибки не правим молча. Сначала читаем traceback, формулируем гипотезу, потом чиним.
5. Каждая версия закрывается коммитом в git с понятным сообщением.
6. После каждой версии — одна-две строки в `NOTES.md`: что было сложно, что повторяется.

## v0. Каркас (1-2 вечера)

**Эмоциональная цель:** открыл `http://localhost:3000` → видишь "FACEIT Clone" и зелёный статус "backend online".

### Что новое узнаёшь
- `venv`: зачем виртуальное окружение, как активировать.
- `git init`, `git add`, `git commit` на базовом уровне.
- `pip install` + `requirements.txt`.
- `npm init`, `package.json`.
- Минимальный FastAPI: что такое app, decorator, uvicorn.
- Минимальный Express: что такое app, route, статика.

### Backend
- Создай `backend/`, активируй venv, поставь `fastapi` и `uvicorn`.
- В `app/main.py` сделай ручку `GET /health` → `{"status": "ok"}`.
- Запусти `uvicorn app.main:app --reload --port 8000`.
- Открой Swagger на `http://localhost:8000/docs`.

### Frontend
- Создай `frontend/`, `npm init -y`, поставь `express`.
- В `server.js`: Express отдаёт статику из `src/` на порту `3000`.
- В `src/index.html`: заголовок "FACEIT Clone" + `<div id="health">checking...</div>`.
- В `src/main.js`: `fetch('http://localhost:8000/health')` → отрисуй "🟢 online" или "🔴 offline".

### Критерии готовности
- `cd backend && uvicorn ...` стартует.
- `cd frontend && node server.js` стартует.
- На `localhost:3000` видишь зелёную точку.
- `git log` показывает минимум 2 коммита.
- В `NOTES.md` записано: что такое venv и зачем он нужен.

### Что узнаёшь как побочный эффект
**CORS.** Когда фронт на `:3000` дергает бэк на `:8000`, браузер заблокирует. Это первая стена. Не лечи через `Access-Control-Allow-Origin: *` бездумно — поставь `fastapi.middleware.cors.CORSMiddleware` и пойми, какие origin разрешаешь. Я объясню при ревью.

## v0.2. Профиль из реального FACEIT API (1-2 вечера)

**Эмоциональная цель:** ввёл свой ник → увидел свой настоящий ELO. **Это первый момент, когда проект становится твой.**

### Что новое узнаёшь
- HTTP-клиент: `httpx` (асинхронный, дружит с FastAPI).
- Переменные окружения через `pydantic-settings` или `os.environ`.
- `.env` файл и `.gitignore` (ключ FACEIT никогда не коммитим).
- Обработка ошибок API: таймаут, 401, 404, 5xx.

### Backend
- Зарегистрируй приложение на `developers.faceit.com`, получи Bearer-token.
- Положи в `backend/.env`: `FACEIT_API_KEY=...`. `.env` в `.gitignore`.
- В `backend/.env.example` оставь пример без значения.
- Сделай `services/faceit_client.py` с функцией `get_player(nickname)`.
- Ручка `/players/{nickname}` теперь зовёт реальный API.
- Если FACEIT вернул 404 — твой бэк тоже 404. Если 401 — 500 с понятным сообщением.

### Frontend
- Без изменений в логике. Только покажи иконку страны (флаг) и аватарку, если приходят.

### Критерии готовности
- Свой ник → свой ELO.
- Удалённый из `.env` ключ → понятная ошибка, не traceback.
- `git status` не показывает `.env` (если показывает — `.gitignore` сломан, исправь).

## v0.4. Детали матча и K/D (2 вечера)

**Эмоциональная цель:** клик по строке матча → раскрывается детальная стата (kills, deaths, ХС%, K/D).

### Что новое узнаёшь
- Pydantic-схема для вложенных объектов.
- Расчёт метрик в отдельной функции (не в роуте!).
- Обработка деления на ноль и отсутствующих полей.
- pytest: первые 2-3 теста на расчёт K/D и headshot rate.

### Backend
- Ручка `GET /matches/{match_id}/stats`.
- Достаёт `player_stats`, считает `kd = kills / max(deaths, 1)`, `hs_rate = headshots / kills * 100`.
- Файл `services/stats.py` с чистыми функциями расчёта.
- `tests/test_stats.py` — 3 теста: норм игрок, 0 deaths, 0 kills.

### Frontend
- Клик по строке → разворачивается блок со статой всех игроков матча.
- Подсветка строки самого MVP (макс score = `kills * 2 - deaths`).

### Критерии готовности
- `pytest` зелёный, минимум 3 теста.
- Хоть один тест ты намеренно сломал, увидел падение, починил.
- Расчёт K/D вынесен из роута в `services/stats.py`.

## v0.5. Кэш в SQLite (2 вечера)

**Эмоциональная цель:** второй запрос мгновенный. На карточке бейдж "🔵 live / ⚪ cached".

### Что новое узнаёшь

### Backend
- Таблица `players_cache`: `nickname, data_json, fetched_at`.
- Логика: при запросе сначала смотрим кэш, если свежее 10 минут — отдаём из БД, иначе идём в FACEIT.
- В ответе добавь поле `source: "cache" | "live"`.

### Frontend
- Маленький бейдж в карточке, показывающий источник.

### Критерии готовности
- Первый запрос медленный, второй мгновенный.
- Через 10 минут — снова медленный.
- Можно удалить файл БД и приложение само создаст её при старте.

### Frontend
- Форма логина/регистрации.
- Токен в `localStorage`.

## v0.9. Матчмейкинг (упрощённый) (3-5 вечеров)

**Эмоциональная цель:** открыл вкладку, нажал "Find Match", открыл вторую вкладку в режиме инкогнито, нажал там тоже — обе вкладки видят "Match found! vs ...". Это первый WebSocket-момент. Кайф большой.

### Что новое узнаёшь
- Что такое WebSocket, чем отличается от HTTP.
- `async` / `await` всерьёз.
- Хранение состояния (очереди) в памяти процесса.
- Простой подбор: первые двое в очереди → матч.

### Backend
- Очередь — обычный `list` или `asyncio.Queue` в памяти.
- WS-эндпоинт `/ws/queue`: клиент подключается → ставится в очередь → как только нашлась пара, обоим отправляется `match_found`.
- Ручка `POST /matches` — создаёт запись о матче в БД.

### Frontend
- Кнопка "Find Match".
- При клике — открывается WebSocket.
- Показывается таймер "ищем... 0:15".
- Когда `match_found` — показывается оппонент и кнопка "Accept".

### Критерии готовности
- Две вкладки → находят друг друга.
- Если закрыть вкладку до подбора — пользователь убирается из очереди.

# Прямо сейчас

1. Создай папку `faceit-clone/` где-нибудь рядом с этим репо.
2. `git init` внутри.
3. Заведи `backend/` и `frontend/` подпапки.
4. Делай v0 шаг за шагом.
5. Когда зелёный `/health` начнёт мигать на странице — пиши `review v0`.

Не пытайся забежать вперёд. Если что-то непонятно — `hint-1`. Не открывай ИИ-чат и не проси написать. Это та точка, в которой прошлые попытки ломались.
